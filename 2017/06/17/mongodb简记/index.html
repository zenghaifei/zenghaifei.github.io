文档数据库mongodb

发表于 2017-06-17   |  

mongdb是一个以文档格式存取的nosql数据库，以其高读写性能、水平扩展性能与模式灵活性著称，适用于性能与灵活性较高的游戏、视频与大数据等领域。

一、特点

  • 1、模式自由, 一个集合中的数据结构可以不同，数据结构由代码控制而不是数据库，灵活性强，在程序开发初始阶段数据结构经常变动的情况下是一个巨大的优势。
  • 2、对象形式存储, 与编程语言中的对象可以简单快速的转换，既避免了复杂低速的关联查询，又避免了关系型数据库需要的异常复杂的OR-Mapping，拥有巨大的开发与服务性能优势
  • 3、高效的水平扩展能力
    关系型数据库水平扩展能力有限，进行分库分表的操作异常麻烦，在面临性能瓶颈时大多会选择垂直扩展（增加数据库服务器内存与服务器性能），由此对单节点的依赖度高，风险较大（即便配备了从库，在节点出现故障时也将大大影响性能）；mongodb并不存在这个问题，
    mongodb支持主从结构的横向集群（对单一节点的依赖度低），从单一的主节点写数据，从一个或多个从节点读数据（当主节点出故障后集群能自动的选出一个次节点作为新的主节点，原来的主节点正常后也将成为集群内的新的次节点，避免了单节点故障的风险）
  • 4、以bson格式存储，一个bson对象大小不超过16M，相较于json提供了额外的存储类型支持(如：Binary data、Timestamp、JavaScript code、Regular expression)
  • 5、对大文件存储提供支持，因二进制对象可存储于bson中，利用GridFs文件存储规范可以存储很大的文件。

    注：GridFS是一个标准，mongodb并没有实现它，利用GridFS进行存储时需要代码实现该存储标准（许多语言都提供了实现的库），该标准的实现方法是将大文件拆分为一个个的小文件以文档的格式存储，并将有关该文件的信息（文件的各个拆分的小文档的依次的id，文件的大小，上传时间，md5等信息）以另外一个元文档的形式存储，该元文档可以有自定义的字段，可以将GridFS当作文件系统来使用。另外，覆盖原来的文件不会真的覆盖，mongodb内部维护了一个version，只是version往前走了
  • 6、一个库可以同时无限制的并发读取
  • 7、对数据的写操作的限制在集合级别或文档级别

    默认的MMAPv1存储引擎使用的是集合级别的锁(一个集合同时只能有一个写操作)，WiredTiger存储引擎使用的是文档级别的锁（一个文档同时只能有一个写操作），在磁盘上以压缩的格式存储数据，读取速度快，占用空间小，更充分的利用了硬件的能力，性能表现也更可预期
  • 8、文档内的操作是原子性的，但涉及到多个文档的更新操作就不支持原子性

二、数据库操作命令（js格式脚本）

I. 筛选操作符

数值范围筛选

  • “$lt“ 代表“小于”，”$lte“代表“小于等于”，如 {"age":{$lte:20}} 表示age小于等于20
  • “$gt“ 代表“大于”，”$gte“ 代表“大于等于”

逻辑选择筛选

  • “$or“ 代表“或”，如{$or:[{'age':{$lt:25}},{'city':'广州'}]}表示age小于25或者city为广州
  • “$and“ 代表“且”
  • “$not“ 代表“非”，但后面跟的是正则表达式
II. 查询操作
  • db.[collectionName].find({})，其中大括号中可以加上查询的筛选条件，不加条件表示不作限制
III. 更新

更新一个键值

（如该文档无该键则自动增加该键）

  • db.<collectionName>.update(<be updated document json filter>, {$set:{<key>:<value>,...}})

删除一个文档的一个或多个键

  • db.<collectionName>.update(<be updated document json filter>,{$unset:{<key>:"",...}})，其中key的值只有格式意义，可以随便指定

给文档数组类型的键添加一个或多个值

  • db.<collectionName>.update(<be updated document json filter>,{$push:{<key>:<value>,...}}，其中为数组时代表同时添加多个值
IV. 删除
  • db.<collectionName>.remove(<be removed document json filter>)代表的是删除指定collection中指定条件的文档，如指定的是一个空json则代表删除该集合中的所有文档，如：删除一个集合：db.<collectionName>.drop()

VI. 内置函数
  • limit限制返回的数量 ,如：db.restaurant.find().limit(10)
  • sort对返回的结果进行排序, 如:db.customer.find().sort({'age':1})表示按年龄进行顺序排序(“1”代表顺序，“-1”代表按倒序排序)
  • skip跳过前面的文档，如：db.restaurant.find().skip(1)代表返回结果省略第一条文档

三、数据库管理

I. 创建用户

user文档字段介绍：

  • user字段，为新用户的名字
  • pwd字段，用户的密码
  • cusomData字段，为任意内容，例如可以为用户全名介绍
  • roles字段，指定用户的角色，可以用一个空数组给新用户设定空角色；在roles字段,可以指定内置角色和用户定义的角色。

Built-In Roles（内置角色）：

  • 1、数据库用户角色：read、readWrite;
  • 2、数据库管理角色：dbAdmin、dbOwner、userAdmin；
  • 3、集群管理角色：clusterAdmin、clusterManager、clusterMonitor、hostManager；
  • 4、备份恢复角色：backup、restore；
  • 5、所有数据库角色：readAnyDatabase、readWriteAnyDatabase、userAdminAnyDatabase、dbAdminAnyDatabase
  • 6、超级用户角色：root //这里还有几个角色间接或直接提供了系统超级用户的访问（dbOwner 、userAdmin、userAdminAnyDatabase）
  • 7、内部角色：__system

验证新用户

  • mongo -u <userName> -p <pwd> --authenticationDatabase <database>
