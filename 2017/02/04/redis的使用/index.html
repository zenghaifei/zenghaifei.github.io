redis的使用

发表于 2017-02-04   |  

启动与连接

redis-cli是一个管理redis的命令行交互工具，使用它便可以直接在命令行中连接操作redis了

-bash中启动redis服务: $ redis-server /usr/local/etc/redis.conf

  • 命令行中关闭redis: $ redis-cli shutdown save|nosave
  • redis-cli连接redis: $ redis-cli -h ipAddress -p port

管理命令

  • 查看按通配符匹配的所有key: KEYS pattern,其中pattern的匹配方式为通配符匹配
  • bgsave 将缓存中的数据以RDB文件的形式持久化到硬盘中（注意其与save命令的差别，前者是不阻塞式的在后台保存，redis依然对外提供服务，后者是阻塞式的保存）
  • 删除当前连接redis库的所有key：$ redis-cli flushdb

基本存储规范

  • key应具有良好的可阅读性
  • key不宜太长，否则存储时耗空间与带宽资源，大大降低效率
  • 用“：”作为key名称的分割符（或“.”、“／”，也偶尔用“|“）
  • 坚持始终一致的一种key命名风格

几种基本存储结构

与存储的结构类型无关的通用key操作与规则：

  • 删除一个键值对： DEL key
  • 查看指定的键是否存在： EXISTS key
  • 查看指定键的存储结构类型：TYPE key
  • 设置一个键值的存续时间（秒）: EXPIRE key seconds
  • 查看一个键的剩余存续时间：TTL key -2代表不存在，-1代表存续时间不受限制，大于零代表还可以存续多长时间
  • 当一个key不存在时，对任何一种类型的key的设置或增加操作redis都将自动创建一个空的该key
    对于任何一种存储结构，Values can be strings (including binary data) of every kind, for instance you can store a jpeg image inside a key. A value can’t be bigger than 512 MB.

字符串String

  • 设置一个key：SET key value
  • 获取一个key的值：GET key
    -将一个存储数值的key自增1: INCR key (相反的指令为decr)
  • 将一个存储数值的key自增指定数值：INCRBY key incrementNum （相反的指令为decrby）
    注意：使用incr命令增加数值与手动执行get加1后再set的好处是：incr操作是原子的，多客户端同时执行该操作也不会有问题，而set操作则会存在并发存取不一致的问题

典型应用场景：统计页面访问量

链表list

  • LRANGE key beginIndex endIndex 查看该list的一个从beginIndex到endIndex的子集 指定为-1代表是到末尾(下标从0开始)
  • LLEN key 查看指定list的长度
  • RPUSH key value 往右（后）添加一个元素
  • LPUSH key value 往左（前）添加一个元素
  • RPOP key 移除最后一个元素，并返回
  • LPOP key 移除第一个元素，并返回
  • BRPOP key timeout 类似于rpop key，差别在于前者在无元素时会等待直到有新元素进入或超时后才会返回，在“生产-消费”模式中大大减少或消除了无效的轮询。
  • LTRIM key beginIndex endIndex 将该list的一个从beginIndex到endIndex的子集作为该key的新的值，抛弃其它元素，对于存储最新的多少条信息非常有用

典型应用场景：“生产-消费”模式、存储最新的一些动态（如社交网络）

无序集合set

  • SADD key value1 [value2..] 增加一个或多个元素到一个set集合中
  • SREM key value1 [value2..] 从一个set集合中移除一个或多个元素
  • SISMEMBER key 验证一个元素是否在集合中
  • SMEMBERS key 查看set集合中的所有成员
  • SUNION key1 [key2..] 联合一个或多个set返回一个整合后的set
  • SPOP key [number]随机的移除一个或多个元素
  • SRANDMEMBER key 获取集合中的一个随机元素而不移除该元素
  • SUNIONSTORE destinationKey key1 [key2..] 将一个或多个集合联合起来存储为另外一个集合
  • SCARD key 查看一个集合有多少元素（cardinality）

入栈即时排序的有序集合zset

  • ZADD key score value 增加一个元素
  • ZRANGE beginIndex endIndex [WITHSCORES] 按顺序列出指定位置范围的所有元素，第一个元素的index为0，最后一个为-1
  • ZRANGEBYSCORE key min max 返回score值在指定范围的所有元素, 如：ZRANGEBYSCORE mykey1 -inf inf 代表score从负无穷到正无穷全部返回
  • ZREMRANGEBYSCORE key min max移除score值在指定范围的元素
  • ZINCRBY key increment member给一个元素的score增加指定值
  • ZRANK key member查看指定元素在set中的位置(0开始)

典型应用场景：实时动态更新的排行榜

“键中键”与面向对象式存储——哈希hash

  • HSET user:1000 name “John Smith”
  • HSET user:1000 email “john.smith@example.com”
  • HINCRBY user:1000 visits 100 将key为“user:1000”的hash的visits属性自增100
  • HKEYS user:1000 列出key为”user：1000“的hash的所有key
  • HVALS user:1000 列出key为”user：1000“的hash的所有values
  • HMGET user:1000 name email 查看一个hash的指定几个域,这里为name与email
  • HMSET user:1000 username zenghaifei birthday 1990-05-14 一次同时设置多个键值对
