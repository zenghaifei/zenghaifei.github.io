正则表达式

发表于 2017-07-08   |  

简介

在字符串的处理中，正则表达式是绕不开的话题，它就是为字符串的处理而生的。

确切的讲，它是用来对字符串进行模式匹配处理的一个语言工具，目前在各个编程语言以及编辑器中均有广泛的支持（不同的处理引擎语法与功能实现上会略有差异）；

对我们程序员而言，你既可以在编辑器中用它来快速的进行文本的查找、替换，也可以在程序中利用它进行简洁高效的字符串校验编辑，熟悉后，你一定会为它所带来的效率上的提升所惊讶的！

常用语法

一、常用匹配符

  • 匹配单个数字: \d (d对应单词digit), 匹配单个非数字：\D
  • 匹配单个常见字符: \w (w对应单词‘word’，含：a-z,A-Z,0-9,_)，匹配单个非字符：\W
  • 匹配单个任意字符：.
  • 匹配空白字符：\s（该空白字符可以是一个space、tab（\t）或者new line （\n）），匹配单个非空白字符: \S

二、基本匹配模型

选择

1、枚举选择
  • [abc12] 选择单个字符：代表该字符可以是a、b、c、1、2中的任意一个
  • (abc|efg) 选择一个字符串：代表该字符串要么是‘abc’要么是’efg’
2、范围选择
  • 在…范围：如[1-9]代表1-9的一个数字，[a-z]代表a-z的一个英文小写数字
  • 不在…范围：如[^0-9]代表一个非数字(与[\D]相同),[^\w]代表一个非常见字符（与[\W]相同）
  • “|”或：如：(abc|def)表示匹配abc或def

重复

  • * 代表该符号前的字符或字符模型可以往后重复0到无数次, 如[0-9]* 可以匹配任何长度的数字，包括长度为0（即不匹配）。
  • + 代表该符号前的字符或字符模型可以往后重复1到无数次，如[0-9]+ 可以匹配1位以上长度的数字，不能1位都不匹配。
  • ? 代表该符号前的字符或字符模型可以重复0次或1次， 如[0-9]?9可以匹配19，也可以匹配9。
  • {1}代表重复的次数为1，{1，3}代表重复的次数为1到3之间, 如：a{1,3}表示a重复1到3次，(ab){1,3}代表ab重复1到3次,(hope){1,}代表“hope”重复一次以上

断言（本身并不占字符空间）

  • ^ 表示一行的开始
  • $ 表示一行的结尾
  • \b 表示字符串的边界, \b = word boundary = between \w and \W
  • \B = non-word boundary = between \w and \w or \W and \W
  • (?=a) = followed by a, which can be any regex
  • (?!a) = NOT followed by a
  • (?<=)

三、字符组捕获与更改

1、捕获

除了匹配字符串是否符合指定的模型之外，我们也希望能提取出字符串中一些有价值的信息以备后续使用，这时正则表达式的字符组捕获功能便非常有用了。基本语法为：分别将匹配串中需要捕获的各个域用括号括起来，需要调用各个捕获域的值时只需用$1、$2、$3…(或\1、\2、\3…)分别对应起括号从左往右的1、2、3…的顺序即可。

打括号的区域都会被当作一组被正则引擎捕获，如果需要避免捕获，则在左括号后加上?:，如(?:abc)

1)、字符串替换时使用“$1、$2…”代表各个捕获的组

如：时间字符串2016-10-02中隐含了年、月、日的信息，我们可以使用正则表达式(\d{4})-(\d{2})-(\d{2})及$1、$2、$3将其中各项的值给提取出来（$1、$2、$3分别代表前面表达式中第1、2、3个括号中匹配的值）。

2)、在正则表达式中使用“\1、\2…”引用前面捕获的组

如：(\w)\1代表两个重复的常见字符，匹配’aa’ ‘bb’等重复字符

2、大小写转换

捕获之后的处理很重要，其中进行字符串的大小写转换就是非常常见的一个操作，正则表达式提供了如下四个转换语法（以正则表达式：”(\s)(\w+)(\s.*`)”为例）：

  • 捕获的字符串域全部字符转大写：..\U..\E..，介于\U到\E之间的所有字符在替换时便都会替换为大写，如上例的表达式：$1\U$2\E$3
  • 捕获的字符串域全部字符转小写：..\L..\E..，介于\L到\E之间的所有字符在替换时便都会替换为小写，如上例的表达式：$1\L$2\E$3
  • 捕获的字符串域的首字符转大写：..\u..，\u后的第一个字符在替换时会替换为大写，如上例的表达式：$1\u$2$3
  • 捕获的字符串域的首字符转小写：..\l..，\u后的第一个字符在替换时会替换为小写，如上例的表达式：$1\l$2$3

四、需注意特性

贪婪匹配

对于重复标识符* 、+、？匹配引擎默认总是会试图优先以尽量多的位数进行匹配，多位数匹配不上才会尝试用更小的位数匹配，如需要启用懒惰模式（即以尽量少的位数进行匹配）则需要在这些标识符后再加上？。

如：用正则表达式<.+>匹配字符串“This is a < EM>first< /EM> test”，匹配的第一个结果将为：“ < EM>first< /EM>”而不是< EM>，因匹配引擎默认优先进行贪婪匹配，将表达式改为<.+?>后改为惰性匹配，此时的第一个匹配结果才是”< EM>”

最左匹配

字符串从左向右找到第一个匹配项便停止

正则引擎的支持度问题

有些正则语法在有的正则引擎中并不支持，如atom编辑器目前对一些正则表达式还无法解析(经测试intelij Idea支持较好)

五、正则表达式在代码中的使用原则（基于执行效率）

  1. 尽量避免贪婪匹配
  2. 尽量使用开始（^）与结束($)断言
  3. 尽量的具体
  4. 对于不必要捕获的括号加上？：
  5. 少使用回溯
  6. 适度精确，避免表达式过于复杂（可读性差，执行效率低）
  7. 对于能用少量的代码就能解决的问题（如10几行代码），使用代码解决
